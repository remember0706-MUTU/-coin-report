## 2. ICT 분석 방법론
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

탑다운 분석 순서 준수:
  1. Market Structure 확인 (BOS / ChoCH / MSS)
  2. Liquidity 분석 (BSL / SSL 위치 파악)
  3. Order Block(OB) 식별
  4. FVG / BPR 확인
  5. 시나리오 A(불리시) / B(베어리시) 구성

딜링 레인지:
  - EQ = (레인지 상단 + 레인지 하단) / 2 반드시 계산
  - 현재가 → Premium / Discount / EQ 근방 분류

ICT 핵심 용어 (최초 등장 시 풀네임 병기):
  Market Structure Shift(MSS), Break of Structure(BOS),
  Change of Character(ChoCH), Fair Value Gap(FVG),
  Balanced Price Range(BPR), Order Block(OB),
  Buy-Side Liquidity(BSL), Sell-Side Liquidity(SSL)

세션별 KST 시간대 참고 (세션 판단 시 이 표 기준으로 명시):
  00:00~02:00 → 뉴욕 세션 후반부 마무리
  02:00~04:00 → 뉴욕 후반 킬존
  04:00~09:00 → 세션 간 과도기 (변동성 최소)
  09:00~14:00 → 아시아 세션
  14:00~15:00 → 런던 대기 구간
  15:00~17:00 → 런던 킬존
  17:00~22:30 → 뉴욕 대기 구간
  22:30~00:00 → 뉴욕 킬존 (Judas Swing 관찰)
